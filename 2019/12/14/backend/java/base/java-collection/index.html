    集合和数组的区别

    1. 数组声明了数据类型，所以数据必须是同一种数据类型；集合可以不声明，所以可以是不同数据类型的集合，如果想要确保是同一种数据类型可以用泛型指定。
    2. 数组的是静态的，长度一旦指定就无法改变；集合可以扩容。
    3. 数组可以存放基本数据类型和引用类型；集合只能存放引用类型。
    4. 数组是java语言中的内置数据类型，执行效率和内置检查都是最快的。
    5. 数组查询速度快，增删速度慢，存储区连续，占用内存空间大。
    6. 链表查询速度慢，增删速度快，存储区散列不连续，占用空间小。

    链接

    List

    ArrayList : 基于数组实现的非线程安全的集合。查询元素快，插入，删除中间元素慢。
    LinkedList : 基于链表实现的非线程安全的集合。查询元素慢，插入，删除中间元素快。
    Vector : 基于数组实现的线程安全的集合。线程同步（方法被synchronized修饰），性能比ArrayList差。
    CopyOnWriteArrayList : 基于数组实现的线程安全的写时复制集合。线程安全（ReentrantLock加锁），性能比Vector高，适合读多写少的场景。

    本人写到这里其实一直心中一直有个问题难以说服自己：为什么要使用数组加链表这种数据结构呢？心中有几种猜想，希望看到的人多指教。

    1. 用数组的原因我觉得就是因为数组查找起来速度快，并且我们删除操作比较少（这时候我们不用数组还有更好的选择吗。。。我感觉要我选择我也选数组。本人上面提到数组语言的内置类型哦，速度超快的哦）。
    2. 用链表的原因，本人觉得HashMap可以不指定数据类型进行插入，但是key的计算要根据hashcode方法进行计算，不同的引用类型的hashcode可能会有相似性，插入数据变多时hash冲突的情况越来越大，但是解决hash冲突有两种方案：开放地址法和链表法（这两种方法本人以后再去介绍）。其中开放地址法缺点太多，速度相比链表法慢，所以选择了链表（总结：就是为了更好的解决hash冲突，并且注意效率）。
    3. 用红黑树的原因：当链表的长度变长之后查询的速度也相应的变慢了（哇java的开发人员可不允许自己一手创建的hashmap性能不好哦），所以JDK1.8他们增加了一个阀值 static final int TREEIFY_THRESHOLD = 8;当链表的长度超过8之后就变成了红黑树，来增加查询的效率。

    java的开发人员可以很精明的哦。本人觉得原因如下：

    1. hash冲突要达到8对于我们平常来说，可能就像买彩票中奖一样的几率会出现吧（当然可能只有本人中奖率低），这时候对于链表来说长度很短查询起来其实也很快，和红黑树不相上下。
    2. 红黑树虽然查询的时候速度快，但是凡是有利就有弊，生成树的结构的时候还有做额外的运算，如果不幸运的话还要左右旋来调节树的结构来达到相对的平衡（红黑树类似于平衡二叉树，但是不用像平衡二叉树似的追求绝对的平衡）

    总结：

    1. hashmap是非线程安全的
    2. hashmap可以put key的值是null，value的值是null，但是hashcode和equals的值一样value的就会被覆盖,所以就只能有一个kay值为null的
    3. hashmap的默认值大小是16，负载因子是0.75，阀值是8
    4. 初始值是2的幂数，扩容值是2的倍数（明天再讲解为什么，明天讲解扩容，包括JVM怎么去通知扩容的）
    5. JDK1.7之前底层是数组加链表，1.8之后变成数组加链表加红黑树

    LinkedHashMap

    LinkedHashMap继承了HashMap,是HashMap的一个子类，底层是用哈希表和双向链表来保存数据的，它保留了插入的顺序，如果想要输入顺序和输出顺序相同的时候就应该选用LinkHashMap。
    基本操作和HashMap类似，它是通过重写HashMap里面的方法实现的。

    LinkedHashMap和HashMap都采用了同样的Hash算法，但是LinkedHashMap重新定义了存放元素的Entry，除了可以存放原先的key、value、hash、和下一个值得引用外，还多了上一个元素和下一个元素得引用。从而在哈希表得基础上形成了双向链表。

    LinkedHashMap实质上调用了HashMap的构造方法生成底层的table数组，但是除此之外还增加了accessOrder这个参数，如果不设置默认为false，表示按照插入顺序迭代；当然也可以设置成true按照访问顺序迭代。

    总结：

    1. LinkedHashMap继承HashMap，基本操作和HashMap类似，但是不同的是底层是哈希表和双向链表。
    2. LinkedHashMap保留了插入的顺序，通过accessOrder参数设置是保留插入顺序还是访问顺序。

    HashMap是线程不安全的，一旦出现多线程访问就会出现数据操作混乱的现象；在不考虑性能方面我们可以考虑HashTable或者Collections.synchronizedMap(hashMap),这两种基本上是对整个hash表结构上锁，这样在锁表期间，其他线程就要进行等待，从而性能变低。
