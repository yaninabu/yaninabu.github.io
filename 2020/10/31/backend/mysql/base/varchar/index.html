    • varchar(10)和varchar(100)存储 2个字符的内容，是否真的没有区别？

    varchar和char比起来，确实是可变长度的。那什么是可变长度呢？答案是在硬盘存储上面是可变的，也就是存储2个字符在硬盘上面varchar(10)和varchar(100)是一样的都占用2个字符的位置。但是在内存的存储上却是不一样的，也就是说当遇到mysql生成临时表(order sort等)的时候，varchar会转成char，转换成char类型的长度就是varchar所占的内存的长度，所以所占用的内存就变大了，那么扫描时间就变长了。

    varchar的最大长度是多少呢？

    varchar的最大长度是65535，但是实际上并不是能存储65535个，实际是上需要1到2个字节存储内容的长度(当小于等于255的时候需要1个字节(2^8只能存储255个)，大于255的时候需要2个)，如果有不可为空的限制的话，需要一个字节存储不可为空的标识。如果没有不可为空的限制则不需要额外的一个字节

    存储最多多少汉字（没有为空限制的情况）

    在utf-8状态下的varchar，最大只能到 (65535 - 2) / 3 = 21844 余 1。
    在gbk状态下的varchar, 最大只能到 (65535 - 2) / 2 = 32766 余 1

    varchar(2)能存多少个汉字和字母

    4.0以下varchar(2)代表两个字节，也就是可以存两个字母，utf-8情况下一个汉字=3个字节，所以一个汉字也存储不了

    5.0以上varchar(2)代表两个字符，也就是可以存储两个字符和两个汉字。只是实际长短不一样而已。
