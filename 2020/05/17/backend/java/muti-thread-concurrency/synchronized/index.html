synchronized

后端/ java/ 多线程与高并发
创建时间:2020-05-17 17:43
阅读:
  1. 对对象上锁
  2. synchronized 允许加锁和不加锁的方法同时执行
  3. 可重入锁

对对象上锁

  1. 上锁对象可以任意指定，利用对象上面的两位值，去判断是否能拿到锁 每次new一个对象很麻烦，我们可以写成 或者可以写成在方法上面加synchronized 或者写成 在static方法上面加了static，相当于锁了T.class。要注意的是不能锁String/Integer/Long等基本数据类型的引用

    synchronized 允许加锁和不加锁的方法同时执行

    如果test()方法也加上synchronize，就变成了一个运行完另一个再运行
    加完锁之后效率低一百倍，如果业务允许脏读，读操作能不锁就不要加锁

可重入锁

必须是可重入锁，不然会造成死锁现象
