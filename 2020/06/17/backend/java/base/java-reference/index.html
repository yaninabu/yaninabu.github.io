引用的分类

强、软、弱、虚

强引用

普通的引用是强引用 MyFinalize myFinalize = new MyFinalize() 一定不会被回收（一个引用指向它）除非myFinalize = null, 将引用去掉

发现永远不会打印finalize

软引用

SoftReference
堆内存不够用了就会回收（-Xms20M -Xmx20M）

作用

主要是用作缓存里面用

弱引用

WeakReference
只要遭遇到GC就会回收；一个对象指向弱引用对象弱引用对象指向另一个对象

作用

只要有一个强引用指向weakReference，只要强引用消失掉就不用管了(GC自己回收掉)，一般用在容器里WeakHashMap

典型应用ThreadLocal

如果tl指向的ThreadLocal消失的时候（tl是局部变量，方法结束的时候tl就消失了），如果是key指向ThreadLocal是强引用的Key指向ThreadLocal一直存在，内存泄露(一直不用一直不会被回收) 不是OOM（内存溢出）
但是还是有问题：ThreadMap指向的key是强引用，虽然是Key指向的ThreadLocal是弱引用不用被回收了，key变成null指向的value值找不到了，但是ThreadMap也会越积攒越多，也会出现内存泄露的问题，所以在用到ThreadLocal的时候，不用了一定一定要执行tl.remove(把某一条key和value remove掉而不是将整个ThreadMap remove掉，当前线程可能还有很多其他的key value键值对)操作

虚引用：

PhantomReference(必须是两个参数，并且第二个参数必须是一个队列)
虚引用就算里面有值也拿不到（不能让你get到为啥让你get到堆外内存，万一你进行了操作，我自己管理不到了怎么办），只是给你一个通知，写虚拟机的人还是能拿到了
管理堆外内存（是给写GVM的人用的，不是给程序员用的）
虚引用，最虚的引用，GC一旦碰到就回收，只是当回收掉了之后，放到一个队列里面去，让你接收到一个通知，什么时候你检测到这个队列里面的有一个值就说名已经被回收掉了，通知的方式就是往队列里面放值，如果你想知道就不断监听
直接内存DirectBuffer不被JVM虚拟机管理，被操作系统直接操作堆外内存，如果DirectBuffer变为空，但是是堆外内存根本就回收不了。
当你写Natty，或者是直接操作操作系统的内存的时候而不是JVM里面能控制的内存的时候，回收堆外内存的时候，我们可以用虚引用监听之后做出相应的处理清除堆外内存（c或者c++delete或者free）java的Unsafe类里面也有堆外内存处理的方法（在1.9之前可以通过反射拿到unsafe类，但是1.9之后不能操作里面的方法了）freeMemory方法

作用

接受通知，对堆外内存进行相应的处理
